Tiro Libre

Objetivo

Tiro Libre es una plataforma para alquilar canchas. Aquí, los propietarios publican sus canchas y los jugadores pueden buscar las mejores opciones para disfrutar con amigos, cuidar el bolsillo, salvar las distancias e incluso crear salas para encontrar nuevos jugadores y formar un equipo ganador. El programa diferencia 2 roles principales:

  • Jugador: Puede alquilar, buscar canchas y armar salas para buscar jugadores.
  • Canchero: Puede publicar y promocionar su cancha, proporcionando información como ubicación, precio y número de teléfono.

Tecnologías Utilizadas

  • HTML
  • CSS
  • JavaScript
  • PHP
  • MySQL
  • Bootstrap
  • Laravel
  • Tailwind CSS

Funciones para Cancheros (Dueños de Canchas)

  • Creación, edición y eliminación de canchas en la plataforma
  • Visualizar los turnos de manera practica desde una agenda y calendario
  • Recibir notificaciones de los jugadores cuando se solicita un turno a través de WhatsApp
  • Llevar un registro de los turnos y poder aceptar o rechazar turnos desde la plataforma (además se evita que los clientes consulten por turnos que no están disponibles lo que agiliza más el negocio)

Funciones para Jugadores

  • Búsqueda de canchas por diferentes criterios
  • Reserva de turnos a través de la plataforma. (se envía una solicitud y esta puede ser aceptada o rechazada lo cual se notificará en WhatsApp)
  • Poder dar una calificación de la cancha
  • Visualización de las canchas
  • Posibilidad de encontrar compañeros de equipo si no se tiene un equipo completo
  • El apartado “Mapa” muestra todos los predios disponibles en el mapa de la ciudad de Gualeguaychú
  • En el apartado “Mis turnos” podrá ver las reservas realizadas por él y las salas a las cual se unió o creó para conseguir jugadores

Objetivo

Checking es un proyecto desarrollado como parte de la materia de "Programación II" en la facultad Sedes Sapientiae. Este sistema tiene como objetivo principal registrar las asistencias de los alumnos de manera eficiente y sencilla.

Tecnologías Utilizadas

  • MySQL
  • HTML
  • CSS
  • JavaScript

Descripción

El sistema permite a los alumnos registrarse, iniciar sesión y registrar su asistencia de manera sencilla. Los profesores pueden gestionar las asistencias de sus alumnos y generar informes de asistencia.

FoodService

Descripción

El sistema fue diseñado para una empresa destinada a servicios de consultoría y desarrollo de software. Permite a los usuarios registrarse, iniciar sesión y realizar pedidos de viandas de manera sencilla. Además, cuenta con una funcionalidad de carga de menús mediante OCR, lo que facilita la gestión de los mismos. Los administradores pueden gestionar los pedidos y los menús de forma eficiente.

Tecnologías Utilizadas

  • HTML
  • CSS
  • JavaScript
  • PHP
  • MySQL
  • AJAX
  • Bootstrap
  • Tesseract OCR

Dificultades

Durante el desarrollo del proyecto, la principal dificultad fue trabajar con una herramienta proporcionada por la empresa, una especie de framework propio que limitaba considerablemente la flexibilidad del código. Este framework no permitía la instalación de paquetes externos ni contaba con una versión base limpia, ya que se partía de proyectos anteriores. Además, la documentación era inexistente y los comentarios en el código no aportaban claridad, lo que hizo más compleja la comprensión de su estructura y funcionamiento. Estas restricciones representaron un desafío particular al momento de implementar ciertas funcionalidades avanzadas. Una de ellas fue la carga de menús mediante reconocimiento óptico de caracteres (OCR). Debido a las limitaciones del entorno, no fue posible utilizar herramientas modernas como npm, por lo que debí integrar Tesseract OCR de forma manual. Esto implicó realizar una configuración cuidadosa y llevar a cabo múltiples pruebas para asegurar su correcto funcionamiento dentro del sistema. A pesar de todas estas dificultades, logré superar los obstáculos y desarrollar un sistema funcional y eficiente, cumpliendo con los requisitos planteados por el proyecto.
